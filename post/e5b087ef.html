This postintroduces three algorithms that are closely related to each other, which are the solution of BOE to get optimal policies.

The algorithms introduced in this chapter are called dynamic programming algorithms

Recall that the Bellman optimality equation：

v=f(v)=max⁡π(rπ+γPπv)v = f(v) = \max_{\pi}(r_{\pi} + \gamma P_{\pi}v) v=f(v)=πmax​(rπ​+γPπ​v)

Value iteration algorithm

The contraction mapping theorem suggests an iterative algorithm:

vk+1=f(vk)=max⁡π(rπ+γPπvk),k=1,2,3,…v_{k+1} = f(v_k) = \max_{\pi}(r_\pi + \gamma P_\pi v_k), \quad k = 1, 2, 3, \ldots vk+1​=f(vk​)=πmax​(rπ​+γPπ​vk​),k=1,2,3,…

where v0v_0v0​ can be arbitrary. This algorithm can eventually find the optimal state value and an optimal policy.

This algorithm is called value iteration. It can be decomposed to two steps.

The value iteration algorithm can be decomposed into two steps:

  • Step 1: Policy Update

    This step is to solve

    πk+1=arg⁡max⁡π(rπ+γPπvk)\pi_{k+1} = \arg \max_{\pi}(r_{\pi} + \gamma P_{\pi} v_k) πk+1​=argπmax​(rπ​+γPπ​vk​)

    where vkv_kvk​ is given.

  • Step 2: Value Update

    vk+1=rπk+1+γPπk+1vkv_{k+1} = r_{\pi_{k+1}} + \gamma P_{\pi_{k+1}} v_k vk+1​=rπk+1​​+γPπk+1​​vk​

Question: Is vkv_kvk​ a state value?

Answer: No, because it is not ensured that vkv_kvk​ satisfies a Bellman equation.

To implement this algorithm, we need to further examine its elementwise form.

Step1: Policy update

The elementwise form of

πk+1=arg⁡max⁡π(rπ+γPπvk)\pi_{k+1} = \arg \max_{\pi}(r_\pi + \gamma P_\pi v_k) πk+1​=argπmax​(rπ​+γPπ​vk​)

is

πk+1(s)=arg⁡max⁡π∑aπ(a∣s)(∑rp(r∣s,a)r+γ∑s′p(s′∣s,a)vk(s′)⏟qk(s,a)),s∈S\pi_{k+1}(s) = \arg \max_{\pi} \sum_a \pi(a|s) \left( \underbrace{\sum_r p(r|s,a) r + \gamma \sum_{s'} p(s'|s,a) v_k(s')}_{q_k(s,a)} \right), \quad s \in S πk+1​(s)=argπmax​a∑​π(a∣s)⎝⎜⎜⎜⎜⎜⎛​qk​(s,a)r∑​p(r∣s,a)r+γs′∑​p(s′∣s,a)vk​(s′)​​⎠⎟⎟⎟⎟⎟⎞​,s∈S

The optimal policy solving the above optimization problem is

πk+1(a∣s)={1a=ak∗(s)0a≠ak∗(s)\pi_{k+1}(a|s) = \begin{cases} 1 & a = a^*_k(s) \\ 0 & a \neq a^*_k(s) \end{cases} πk+1​(a∣s)={10​a=ak∗​(s)a=ak∗​(s)​

where ak∗(s)=arg⁡max⁡aqk(a,s)a^*_k(s) = \arg \max_a q_k(a,s)ak∗​(s)=argmaxa​qk​(a,s). πk+1\pi_{k+1}πk+1​ is called a greedy policy, since it simply selects the greatest q-value.

Step2: Value update

The elementwise form of

vk+1=rπk+1+γPπk+1vkv_{k+1} = r_{\pi_{k+1}} + \gamma P_{\pi_{k+1}} v_k vk+1​=rπk+1​​+γPπk+1​​vk​

is

vk+1(s)=∑aπk+1(a∣s)(∑rp(r∣s,a)r+γ∑s′p(s′∣s,a)vk(s′)),s∈Sv_{k+1}(s) = \sum_a \pi_{k+1}(a|s) \left( \sum_r p(r|s, a)r + \gamma \sum_{s'} p(s'|s, a)v_k(s') \right), \quad s \in \mathcal{S} vk+1​(s)=a∑​πk+1​(a∣s)(r∑​p(r∣s,a)r+γs′∑​p(s′∣s,a)vk​(s′)),s∈S

Since πk+1\pi_{k+1}πk+1​ is greedy, the above equation is simply

vk+1(s)=max⁡aqk(a,s)v_{k+1}(s) = \max_a q_k(a, s) vk+1​(s)=amax​qk​(a,s)

Procedure summary:

vk(s)→qk(s,a)→greedy policy πk+1(a∣s)→new value vk+1=max⁡aqk(s,a)v_{k}(s) \rightarrow q_{k}(s,a) \rightarrow \text{greedy policy } \pi_{k+1}(a|s) \rightarrow \text{new value } v_{k+1} = \max_{a} q_{k}(s,a) vk​(s)→qk​(s,a)→greedy policy πk+1​(a∣s)→new value vk+1​=amax​qk​(s,a)
