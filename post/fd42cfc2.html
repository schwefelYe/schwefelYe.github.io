Reward

Reward：A real number we get after taking an action.

  • A Positive reward represents encouragement to take such actions.
  • A negative reward represents punishment to take such actions.
  • Can positive mean punishment？Yes.

In the grid world example, the rewards are designed as follows:

  • If the agent attempts to exit the boundary, let rboundary=−1r_{boundary} = −1rboundary​=−1.
  • If the agent attempts to enter a forbidden cell, let rforbidden=−1r_{forbidden} = −1rforbidden​=−1.
  • If the agent reaches the target state, let rtarget=+1r_{target} = +1rtarget​=+1.
  • Otherwise, the agent obtains a reward of rother=0r_{other} = 0rother​=0.

A reward can be interpreted as a human-machine interface, with which we can guide the agent to behave as we expect.

Trajectories, returns and episode

Trajectory and returns

A trajectory is a state-action-reward chain

s1→r=0a2s2→r=0a3s5→r=0a3s8→r=1a2s9.s_{1} \xrightarrow[r=0]{a_{2}} s_{2} \xrightarrow[r=0]{a_{3}} s_{5} \xrightarrow[r=0]{a_{3}} s_{8} \xrightarrow[r=1]{a_{2}} s_{9} . s1​a2​r=0​s2​a3​r=0​s5​a3​r=0​s8​a2​r=1​s9​.

The return of this trajectory is defined as the sum of all the rewards collected along the trajectory:

return=0+0+0+1=1.\mathrm{return}=0+0+0+1=1. return=0+0+0+1=1.

Returns can be used to evaluate policies.

Discounted return

A trajectory may be infinite:

s1→a2s2→a3s5→a3s8→a2s9→a5s9→a5s9→a5s9…s_1\xrightarrow{a_{2}}s_{2}\xrightarrow{a_{3}}s_{5}\xrightarrow{a_{3}}s_{8}\xrightarrow{a_{2}}s_{9}\xrightarrow{a_{5}}s_{9}\xrightarrow{a_{5}}s_{9}\xrightarrow{a_{5}}s_{9}\ldots s1​a2​​s2​a3​​s5​a3​​s8​a2​​s9​a5​​s9​a5​​s9​a5​​s9​…

The return is

return=0+0+0+1+1+1+⋯=∞,\mathrm{return}=0+0+0+1+1+1+\cdots=\infty, return=0+0+0+1+1+1+⋯=∞,

The definition is invalid since the return diverges

Need to introduce a discount rate γ∈(0,1)\gamma\in(0,1)γ∈(0,1)

  1. the sum becomes finite
  2. balance the far and near future rewards:
    • if γ\gammaγ is close to 0, the value of the discounted return is dominated by the rewards obtained in the near future.
    • if γ\gammaγ is close to 1, the value of the discounted return is dominated by the rewards obtained in the far future.

discounted return:

Episode

When interacting with the environment by following a policy, the agent may stop at some terminal states. The resulting trajectory is called an episode (or a trial).

Example：episode

s1→a2s2→a3s5→a3s8→a2s9s_{1}\xrightarrow{a_{2}}s_{2}\xrightarrow{a_{3}}s_{5}\xrightarrow{a_{3}}s_{8}\xrightarrow{a_{2}}s_{9} s1​a2​​s2​a3​​s5​a3​​s8​a2​​s9​

An episode is usually assumed to be a finite trajectory. Tasks with episodes are called episodic tasks.

some tasks may have no terminal states, meaning that the process of interacting with the environment will never end. Such tasks are called continuing tasks

we can treat episodic and continuing tasks in a unified mathematical manner by converting episodic tasks to continuing ones.

  • Option 1：we can treat the terminal state as a special absorbing state, meaning that the agent never leaves a state once reached.
  • Option2：treat the terminal state as a normal state with a policy, the agent can still leave the target state and gain r=+1r = +1r=+1 when entering the target state.
